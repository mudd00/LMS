## 기능 요구사항 (Functional Requirements)

### 1. 회원 시스템

#### 1.1 인증
- **FR-1.1.1** 이메일 기반 회원가입
  - 이메일, 사용자명, 비밀번호 입력
  - 이메일 중복 검증
  - 비밀번호 암호화 저장 (BCrypt)
  - 기본 권한: USER
- **FR-1.1.2** 로그인
  - 이메일/비밀번호 인증
  - JWT 토큰 발급 (24시간 유효)
- **FR-1.1.3** 소셜 로그인 (선택)
  - Google, Kakao 연동

#### 1.2 권한 시스템
- **FR-1.2.1** 권한 레벨
  - USER: 일반 사용자
  - GM (Game Master): 관리자
- **FR-1.2.2** GM 권한
  - 전용 UI 및 기능 제공
  - 사용자 관리 (강제 퇴장, 계정 정지 등)
  - 게임 방 관리
  - 공지사항 작성

#### 1.3 프로필 관리
- **FR-1.3.1** 프로필 조회
  - 자신 및 다른 사용자 프로필 보기
  - 사용자명, 프로필 이미지, 상태 메시지
  - 미니게임 전적 탭 (선택사항)
- **FR-1.3.2** 프로필 수정
  - 프로필 이미지 업로드
  - 상태 메시지 변경
  - 사용자명 변경 (제한적)

### 2. 3D 가상 공간
#### 2.1 메인 광장 (Main Plaza)
- **FR-2.1.1** 3D 광장 렌더링
  - Three.js 기반 3D 환경
  - 실시간 조명 및 그림자 효과
- **FR-2.1.2** 다중 사용자 표시
  - 같은 공간에 있는 다른 사용자 캐릭터 실시간 표시
  - 최대 수용 인원: 100명
- **FR-2.1.3** 접속자 목록
  - Tab 키로 현재 공간(광장 또는 로컬 방) 접속자 목록 모달 토글
  - 활성화 상태에서 Tab/ESC/닫기 버튼으로 닫기
  - 접속 중인 사용자 목록 및 상태 표시


#### 2.2 캐릭터 시스템
- **FR-2.2.1** 캐릭터 생성
  - 로그인 시 3D 캐릭터 자동 생성 (BaseCharacter.gltf)
  - 지정된 스폰 좌표에서 생성
- **FR-2.2.2** 캐릭터 표시
  - 닉네임: 캐릭터 머리 본(정수리) 위에 표시
  - 닉네임 배경: 검은색, 투명도 50%
- **FR-2.2.3** 캐릭터 이동
  - WASD/방향키로 이동
  - Shift 키로 달리기
  - 터치 컨트롤 (모바일)
- **FR-2.2.4** 캐릭터 애니메이션
  - 걷기, 뛰기, 대기 애니메이션
  - 이모티콘 애니메이션
- **FR-2.2.5** 캐릭터 상호작용 범위
  - 다른 캐릭터 클릭 가능 거리: 0.5m 이내
  - 채팅 말풍선 표시 거리: 화면 내 플레이어 기본, 설정에서 범위 조절 가능

#### 2.3 로컬 방
- **FR-2.3.1** 방 생성
  - 사용자의 GPS 위치 기반 방 생성
  - 방 이름, 설명, 테마 설정
  - 비밀번호 설정 기능 제공 (선택)
  - 최대 수용 인원: 24명
  - GPS 권한 필수 요구
  - 1인당 생성 가능한 방 개수: 1개
- **FR-2.3.2** 방 접속
  - 근처 방 검색 (반경 5km)
  - 방 목록 조회 및 입장
  - 방 생성 위치 기준 반경 5km 이내 사용자만 입장 가능
  - 비밀번호가 설정된 방은 비밀번호 입력 후 입장
  - 광장 내 랜덤 위치 스폰
- **FR-2.3.3** 지도 연동
  - Kakao Maps/Google Maps API 활용
  - 생성된 방을 지도에 마커로 표시
  - 현재 위치와 거리 표시
- **FR-2.3.4** 방 관리
  - 방 생성자 권한: 방 삭제, 설정 수정, 사용자 강제 퇴장
  - 방 설정 수정: 이름, 설명, 최대 인원, 비밀번호
  - 로컬 방에서 광장으로 복귀 기능
- **FR-2.3.5** 공간 전환
  - 지도에서 로컬 방 선택 시 해당 공간으로 이동
  - 로컬 방에서 나가기 버튼으로 광장 복귀
  - 전환 시 로딩 화면 표시
  - 다른 로컬 방으로 직접 이동 불가 (광장 경유 필수)

### 3. 소셜 기능

#### 3.1 실시간 채팅
- **FR-3.1.1** 공간별 채팅
  - 광장: 광장 내 전체 채팅
  - 로컬 방: 해당 방 내부 전용 채팅
  - 현재 접속한 공간의 채팅만 표시 (다른 공간 채팅 불가)
  - WebSocket 기반 실시간 메시지
- **FR-3.1.2** 1:1 채팅 (DM)
  - 친구와 개인 메시지
  - 모든 공간에서 접근 가능
  - WebSocket 기반 실시간 메시지
- **FR-3.1.3** 이모티콘
  - 기본 이모티콘 세트 제공
  - 이모티콘 구매 (상점)
  - 채팅에 이모티콘 사용

#### 3.2 친구 시스템
- **FR-3.2.1** 친구 추가
  - 사용자명으로 검색
  - 친구 요청 전송
  - 요청 수락/거절
- **FR-3.2.2** 친구 관리
  - 친구 목록 조회
  - 친구 삭제
  - 온라인 상태 표시
- **FR-3.2.3** 사용자 상호작용
  - 다른 캐릭터 클릭 시 프로필 보기
  - 친구 추가 요청
  - 게임 초대

#### 3.3 차단 및 뮤트
- **FR-3.3.1** 사용자 차단
  - 특정 사용자 차단
  - 차단된 사용자의 채팅 메시지 숨김
  - 차단 목록 관리
- **FR-3.3.2** 채팅 뮤트
  - 특정 사용자 채팅 뮤트
  - 뮤트 목록 관리

### 4. 미니게임 시스템

#### 4.1 게임 로비 생성
- **FR-4.1.1** 게임 방 생성

- **FR-4.1.2** 게임 방 관리

  - 참가자 목록 표시
- **FR-4.1.3** 게임 공간 제한
  - 광장에서 생성한 게임: 광장 내 사용자만 참여 가능
  - 로컬 방에서 생성한 게임: 해당 로컬 방 내 사용자만 참여 가능
  - 다른 공간의 게임 참여 불가
- **FR-4.1.2** 게임 방 관리
  - 방장 권한: 게임 시작, 강제 퇴장, 방 삭제
  - 참가자 목록 표시

#### 4.2 게임 참가

#### 4.3 게임 전적 시스템 (선택사항)
- **FR-4.3.1** 전적 기록
  - 게임별 승/패/무 기록
  - 총 플레이 횟수, 승률
- **FR-4.3.2** 전적 조회
  - 프로필의 '미니게임 전적' 탭에서 통계 제공
  - 게임별 상세 전적
  - 최근 플레이 기록

## 비기능 요구사항 (Non-Functional Requirements)

### 1. 성능 (Performance)

#### 1.1 응답 시간
- **NFR-1.1.1** API 응답 시간 500ms 이하 (평균)
- **NFR-1.1.2** 3D 렌더링 60fps 유지 (데스크톱)
- **NFR-1.1.3** 3D 렌더링 30fps 이상 (모바일)

#### 1.2 동시 접속
- **NFR-1.2.1** 동시 접속자 1,000명 지원
- **NFR-1.2.2** 메인 월드 당 최대 100명 수용
- **NFR-1.2.3** 로컬 월드 당 최대 24명 수용

#### 1.3 최적화
- **NFR-1.3.1** 3D 모델 LOD(Level of Detail) 적용
- **NFR-1.3.2** 캐릭터 인스턴싱으로 다중 사용자 렌더링 최적화

## 관리자 시스템 요구사항 (Admin/Developer Web)

### 목적
개발자(DEVELOPER) 역할을 가진 사용자가 웹 기반 관리 페이지를 통해 서비스 전반을 운영/관리할 수 있도록 지원

### 접근 제어
- **FR-ADMIN-0.1** 역할 기반 리다이렉트
  - 로그인 시 `role === 'DEVELOPER'` → `/admin` 자동 이동
  - 로그인 시 `role === 'USER'` → `/game` (메타플라자)
  - 일반 사용자의 `/admin` 접근 차단 (403 Forbidden)
  - 관리자 IP 화이트리스트 (선택사항)

---

### 1. 대시보드
- **FR-ADMIN-1.1** 주요 지표 요약
  - 실시간 접속자 수
  - 오늘 신규 가입자 / 총 사용자 수
  - 미처리 신고 건수 (배지 표시)
  - 오늘 매출 / 월 매출
  - 활성 게임 방 수 (광장/로컬 방 구분)
  - 최근 에러 알림

---

### 2. 사용자 관리
- **FR-ADMIN-2.1** 사용자 목록 조회
  - 검색: 이메일, 사용자명
  - 필터: 역할(USER/DEVELOPER), 상태(정상/정지), 가입일 범위
  - 정렬: 가입일, 마지막 접속일

- **FR-ADMIN-2.2** 계정 제재
  - 계정 정지/해제
  - 정지 기간 설정 (일시: 1일/7일/30일, 영구)
  - 정지 사유 필수 입력
  - 제재 이력 기록

- **FR-ADMIN-2.3** 역할 변경
  - USER ↔ DEVELOPER 역할 변경
  - 변경 이력 로그 기록

---

### 3. 프로필 관리
- **FR-ADMIN-3.1** 프로필 이미지 검열
  - 전체 사용자 프로필 이미지 그리드 조회
  - 부적절한 이미지 삭제
  - 삭제 시 사용자에게 경고 메시지 발송
  - 반복 위반 시 계정 정지 조치

- **FR-ADMIN-3.2** 프로필 테두리 관리
  - 테두리 에셋 업로드 (PNG/SVG, 권장 크기 500x500px)
  - 테두리 목록 조회/수정/삭제
  - 가격 설정 (무료/유료)
  - 활성화/비활성화 토글
  - 미리보기 기능

### 4. 공지사항 관리

  - 중요 공지 고정(pin) 설정
  - 예약 발행 기능 (날짜/시간 지정)
  - 공지사항 카테고리 (업데이트/점검/이벤트/긴급)

- **FR-ADMIN-4.2** 공지사항 관리
  - 목록 조회 (최신순/고정순)
  - 수정/삭제
  - 발행 취소
